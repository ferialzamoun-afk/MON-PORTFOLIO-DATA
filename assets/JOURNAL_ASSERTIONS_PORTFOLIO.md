# Journal des assertions - Revue portfolio

## Objectif

Suivre, page par page, les assertions verifiees pendant la revue du portfolio GitHub Pages.

Chaque assertion doit recevoir deux statuts :

- **Statut realisation** : `A faire`, `En cours`, `Fait`
- **Statut validation** : `OK`, `NOK`, `KO`

Definitions :

| Statut | Definition |
|---|---|
| `A faire` | Assertion identifiee mais pas encore traitee. |
| `En cours` | Correction, verification ou preuve en cours. |
| `Fait` | Action terminee. |
| `OK` | Assertion validee, preuve coherente. |
| `NOK` | Assertion partiellement validee ou a completer. |
| `KO` | Assertion fausse, lien casse, chiffre incorrect ou preuve absente. |

## Mode d'utilisation

1. Ajouter une ligne par assertion donnee pendant la revue.
2. Garder un identifiant stable : `A001`, `A002`, `A003`, etc.
3. Renseigner la page concernee : accueil, P13, Partie 1, Partie 2, projet P6, veille, etc.
4. Utiliser `OK` seulement si la preuve est visible et coherente.
5. Utiliser `KO` si l'assertion doit etre corrigee avant soutenance.
